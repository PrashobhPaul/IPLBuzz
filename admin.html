🔐 Security notes

Password is checked client-side for simple UX. After unlock, the admin signs in anonymously to Firebase so writes still require auth != null per your DB rules. Change the password by editing ADMIN_PASSWORD near the top of this file's script.

Firebase rules (keep as-is):
{ "rules": { ".read": true, ".write": "auth != null" } }

Also enable Anonymous as a sign-in provider in Firebase Console → Authentication → Sign-in method.
